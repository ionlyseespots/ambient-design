Features

HTML5 sectioning elements within Ambient define themselves, creating a visual structure for the document outline.
This means you can build content into simple or complex layouts.

LESS

Built with LESS to write well-structured, extensible code which is easy to maintain.

Flexbox

A very basic flexbox layout and style with a clean separation of content and presentation.

Patterns

A collection of UI patterns using consistent and conflict-free namespaces.

Orientation

Portrait & Landscape views make Ambient ideal for both responsive web design and adaptive content.


Background

Ambient makes use of HTML5 elements & CSS3 properties to layout semantic markup. While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Sometimes I have a UI that plays well with a traditional (4, 6, 8, and 12 column) grid paradigm but sometimes I need a more complex grid. This means learning classes for structure with floats, columns and media queries. I liked reusing classes for design. Other times I became frustrated reusing classes for structure that were not very clear. I wanted to create a methodology for layout for the unpredictable. The layout should be dynamic, and fit any range of viewports.
