Ambient is a methodology for creating layout.

Ambient has a base grid system for HTML as a presentation format without the need for structure based classes.

There are 4 distinct design patterns when using Ambient.

UI Elements

A set of design patterns for the structure of individual HTML elements, attributes & values.

UX Shells

A set of design patterns for the structure of HTML section elements, attributes & values.

Navs

A set of design patterns for the structure of navigation elements, attributes & values.

Form States

A set of design patterns for the structure of form elements, attributes & values.

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS).

To do that, I believe markup needs to become the presentation. Or more specifically, presentation format. While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the base grid system.

Our goal is to deliver a lean, lightweight base grid system for HTML as a presentation format, while also maintaining a separation from content that adapts across different devices. We also extend HTML’s elements, attributes and values into design patterns to help you display your application’s UI clearly.

Ambient makes use HTML to convey structure in a consistent base grid system, without the need for structure based classes.

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. And it's the key to unlocking this new approach.
