UI patterns

Overview

Attribute selectors | Initial patterns | Components | Layouts | Navs | Forms

DO use UI patterns to layout content.

DO use class names to handle styling elements & content.

Do NOT use class names as a replacement for semantic layout.

UI patterns are built on Ambient's semantic layout.

While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Introduction

UI patterns are layout-related attributes for content.

Attribute selectors

It is possible to style HTML elements that have specific attributes and values, not just class and id.

We define our own namespaces.

am-layout

am-component

am-nav

am-form

Because we define our own custom attribute, we can prescribe base UI patterns.

Matching UI patterns is something I am working on. This is when a base UI pattern makes use of the same attribute and value as an initial pattern.

Chaining UI patterns is in the beginning stages. This is when a base UI pattern value assigns its value as an attribute for the next value. All you need to do is make sure that your attributes return to a value (up to a base UI pattern value).

DO chain down a base UI pattern value.

Do NOT chain more than 3 levels deep.

  • am-component="grid"
    • am-grid="block"
      • am-block="three-fourths"

Summary

Attribute selectors select an element using the presence of a given attribute or attribute value.

[attr]
Represents an element with an attribute name of attr.
[attr=value]
Represents an element with an attribute name of attr and whose value is exactly "value".
[attr~=value]
Represents an element with an attribute name of attr whose value is a whitespace-separated list of words, one of which is exactly "value".
[attr|=value]
Represents an element with an attribute name of attr. Its value can be exactly “value” or can begin with “value” immediately followed by “-” (U+002D). It can be used for language subcode matches.
[attr^=value]
Represents an element with an attribute name of attr and whose value is prefixed by "value".
[attr$=value]
Represents an element with an attribute name of attr and whose value is suffixed by "value".
[attr*=value]
Represents an element with an attribute name of attr and whose value contains at least one occurrence of string "value" as substring.

Initial patterns

Initial patterns are margin & padding and can be added to base UI patterns. Base UI patterns are: am-component, am-layout, am-nav, am-form.

Margin

Initial margin is 0

To add space outside, add the attribute am-margin to any base UI pattern.

If needed, set a specific direction or size with the attribute and value as

am-margin

am-margin="top"

am-margin="right"

am-margin="bottom"

am-margin="left"

am-margin="small"

am-margin="medium"

am-margin="large"

Components

A set of UI patterns for the layout of section elements, attributes & values. The base UI pattern am-component is the basic building block for a section of content.

Grids

Basic grid

To create a basic grid with the base component UI pattern, add the attribute and value am-component="grid" to any section. On orientation portrait, base grid columns do NOT automatically stack.

Base grid rows group blocks on one horizontal line. Base grid blocks divide content into equal sections (blocks).

Basic

To add a row, chain the value grid as the attribute and value am-grid="row" to any div.

To add a block, chain the value grid as the attribute and value am-grid="block" to any div.

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Our goal is to deliver semantic layout for HTML elements, attributes and values, while also maintaining a separation from content that scales up from Portrait to Landscape.

Ambient layout makes use of HTML5 elements & CSS3 properties to layout semantic markup.

Ambient UI patterns are layout-related attributes for content.
