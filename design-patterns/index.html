There are 4 distinct design patterns for layout when using Ambient.

Design patterns are built on Ambient's core fluid design system for HTML sections as a presentation format without the need for structure based classes.

CSS attribute selectors do not care if it is a predefined or a custom namespace. And it's the key to unlocking this new approach.

We define custom namespaces for design patterns. To avoid collisions with other namespaces, we are prefixing default design patterns with "ambient".

Introduction

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. There are 4 distinct design patterns for layout.

  1. UI Elements
  2. UX Shells (first priority)
  3. Navs
  4. Form States (second priority)

Form States

Inline Form

To create an inline form with the default form state design pattern, add the attribute ambientformstate to any form.

Inline

<form ambientformstate>

Stacked Form

To create a default inline form, add the attribute ambientformstate = stacked to any form.

Stacked

<form ambientformstate="stacked">

Aligned Form

To create an aligned form add the attribute ambientformstate = aligned to any form.

Aligned

<form ambientformstate="aligned">

Multi-column Form

To create a multi-column form add the attribute ambientformstate = multicolumn to any form.

Multi-column

<form ambientformstate="multicolumn">

Mix & match form states

To mix & match form states add the attribute ambientformstate = value for the design patterns you want to mix & match to any form.

Stacked + Aligned

<form ambientformstate="stacked aligned">

Multi-column + Stacked

<form ambientformstate="multicolumn stacked ">

Multi-column + Aligned

<form ambientformstate="multicolumn aligned ">
