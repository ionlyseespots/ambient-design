UI Patterns

Overview

Attribute selectors | Initial patterns | Components | Layouts | Navs | Forms

DO use UI Patterns to present content.

Do NOT use UI Patterns as a replacement for semantic layout.

UI patterns are built on Ambient's semantic layout.

While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Introduction

UI patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values.

Attribute selectors

It is possible to style HTML elements that have specific attributes and values, not just class and id.

We define our own namespaces.

am-component

am-layout

am-nav

am-form

Because we define our own custom attribute, we can prescribe base UI patterns without needing to use or duplicate class names for layout.

Built with LESS to write well-structured, extensible code which is easy to maintain.

Matching UI patterns is something I am working on. This is when a base UI pattern makes use of the same attribute and value as an initial pattern.

Chaining UI patterns is in the beginning stages. This is when a base UI Pattern value assigns its value as an attribute for the next value. All you need to do is make sure that your attributes return to a value (up to a base UI Pattern value), and you can chain as much as you want.

  • am-layout="gutter"
    • am-gutter="margin"

Summary

Attribute selectors select an element using the presence of a given attribute or attribute value.

[attr]
Represents an element with an attribute name of attr.
[attr=value]
Represents an element with an attribute name of attr and whose value is exactly "value".
[attr~=value]
Represents an element with an attribute name of attr whose value is a whitespace-separated list of words, one of which is exactly "value".
[attr|=value]
Represents an element with an attribute name of attr. Its value can be exactly “value” or can begin with “value” immediately followed by “-” (U+002D). It can be used for language subcode matches.
[attr^=value]
Represents an element with an attribute name of attr and whose value is prefixed by "value".
[attr$=value]
Represents an element with an attribute name of attr and whose value is suffixed by "value".
[attr*=value]
Represents an element with an attribute name of attr and whose value contains at least one occurrence of string "value" as substring.

Initial Patterns

Initial patterns are margin & padding and can be added to base UI patterns. Base UI Patterns are: am-component, am-layout, am-nav, am-form.

Margin

Initial margin is 0

To add space outside, add the attribute am-margin to any base UI pattern.

If needed, set a specific direction or size with the attribute and value as

Components

A set of UI patterns for the layout of section elements, attributes & values. The base UI Pattern am-component is the basic building block for components.

Grid

To create a grid with the base component UI pattern, add the attribute and value am-component="grid" to any section. On orientation portrait, base grid columns do NOT automatically stack.

To add a row, chain the value grid as the attribute and value am-grid="row" to any div.

To add a block, chain the value grid as the attribute and value am-grid="block" to any div.

Basic grid

Base grid rows group blocks on one horizontal line. Base grid blocks divide content into equal sections (blocks).

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). Ambient avoids CSS classes in the HTML markup itself. While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Our goal is to deliver a semantic layout for HTML elements, attributes and values, while also maintaining a separation from content that adapts across different devices.

Ambient makes use of HTML5 elements & CSS3 properties to convey a consistent layout.

UI patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. And it's the key to unlocking this new approach.
