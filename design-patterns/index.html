Overview

There are 4 distinct design patterns for layout when using Ambient.

Design patterns are built on Ambient's base fluid design system for HTML sections as a presentation format without the need for structure based classes.

CSS attribute selectors do not care if it is a predefined or a custom namespace. And it's the key to unlocking this new approach.

We define custom namespaces for design patterns. To avoid collisions with other namespaces, we are prefixing default design patterns with "ambient".

Introduction

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. There are 4 distinct design patterns for layout.

  1. UI elements
  2. UX Shells (first priority)
  3. Navs
  4. Form states (second priority)

Inline form

To create a default inline form with the base form state design pattern, add the attribute ambientformstate to any form. 

<form ambientformstate>
    <label for="inline-email">Email</label>
    <input id="inline-email" type="email" placeholder="Email">

    <label for="inline-password">Password</label>
    <input id="inline-password" type="password" placeholder="Password">

    <label for="inline-remember">
        <input id="inline-remember" type="checkbox"> Remember me
    </label>

    <button type="submit" class="btn btn-primary">Sign in</button>
</form>

Stacked form

To create a stacked form, add the attribute ambientformstate = stacked to any form. 

<form ambientformstate="stacked">
    <label for="stacked-email">Email</label>
    <input id="stacked-email" type="email" placeholder="Email">

    <label for="stacked-password">Password</label>

    <label for="stacked-remember">
        <input id="stacked-remember" type="checkbox"> Remember me
    </label>

    <button type="submit" class="btn btn-primary">Sign in</button>
</form>

Aligned form

To create an aligned form add the attribute ambientformstate = aligned to any form.

Wrap a <span> around the <label> + <input> elements you want to align. 

<form ambientformstate="aligned">
    <span>
        <label for="aligned-email">Email</label>
        <input id="aligned-email" type="email" placeholder="Email">
    </span>
    <span>
        <label for="aligned-password">Password</label>
        <input id="aligned-password" type="password" placeholder="Password">
    </span>
    <span>
        <label for="aligned-remember">
            <input id="aligned-remember" type="checkbox"> Remember me
        </label>
    </span>
    <button type="submit" class="btn btn-primary">Sign in</button>
</form>

Multi-column form

To create a multi-column form add the attribute ambientformstate = multicolumn to any form.

Wrap a <div> around the <label> + <input> elements you want to column. The amount of <div> define the amount of columns.

    <div>
        <label for="multicolumn-email">Email</label>

Ambient design is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). To do that, I believe markup (HTML) needs to become the presentation format.

Ambient is about HTML as a presentation format.

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. And it's the key to unlocking this new approach.

This site is built with Ambient v1.
