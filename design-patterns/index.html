Layout attributes

Introduction

Ambient is layout

Ambient makes use of CSS3 attributes on elements for page layout.

We define our own namespaces

Because we define our own custom attributes, we can define extendable properties common to all HTML elements as well as HTML5 semantic elements.

Do use CSS3 attributes to layout defined content.
Do NOT use CSS3 attributes as a replacement for styling content better suited to a class name.

Namespaces

Global

Global attributes are layout properties common to all HTML elements. These attributes can be used on any element.

am-margin
am-padding
am-layer

Root

Define your layout. The am-layout attribute must be used on a root HTML5 semantic element, though the attribute may have no effect on some elements. Once you've defined the root element, you can add additional am-layout properties as well as extend child attributes am-layout_SOMETHING to your scaffolding to gradually build a hierarchy that defines your layout.

am-layout

Global

Global layout attributes are layout properties common to all HTML elements. These attributes can be used on any element.

Margin

Default margin is 0
To add space outside an element, add the attribute am-margin to any element.

If needed, specific direction or size is available as:

  • am-margin="top"
  • am-margin="right"
  • am-margin="bottom"
  • am-margin="left"
  • am-margin="small"
  • am-margin="large"

Padding

Default padding is 0
To add space inside an element, add the attribute am-padding to any element.

If needed, specific direction or size is available as:

  • am-padding="top"
  • am-padding="right"
  • am-padding="bottom"
  • am-padding="left"
  • am-padding="small"
  • am-padding="large"

Layers

Default layer is 0
To add depth to an element, add the attribute am-layer="VALUE" to any element.

A potential use case for layers is messaging and alerts. Assign depth values as priorities for: info, success, danger & warning.

Layer depth values are available as

  • am-layer="z-1"
  • am-layer="z-2"
  • am-layer="z-3"
  • am-layer="z-4"
  • am-layer="z-5"

Linear layout

The HTML5 article element aligns explicit sections in a single direction, vertically or horizontally. You can specify a vertical layout direction for explicit sections by adding the attribute and value am-layout="vertical" to the HTML5 article element. The default orientation is horizontal.

Grid layout

Grid layout is used to structure content, and thus relies on the root HTML5 semantic element for intent. The default orientation on portrait for grid layout does not stack.

Grid layout with equal widths

To create a grid layout add the attribute and value am-layout="grid" to any HTML5 semantic element.

<section am-layout="grid">

To add a row, add the attribute and value am-layout="row" to a child div.
To add a column, add the attribute and value am-layout="column" to a child div.

Form layout

Form layout is used to structure forms, and thus relies on the root HTML5 semantic form element for intent.

Example form

Example

<form>

Code 


<form>
    <label for="example-email">Email</label>
    <input id="example-email" type="email" placeholder="Email">

    <input id="example-password" type="password" placeholder="Password">

    </label>

    <button type="submit">Sign in</button>
</form>

Inline form

To create an inline form with the base form state UI pattern, add the attribute am-form to any form.

Inline

<form am-form>

Code 


<form am-layout="inline">
    <label for="inline-email">Email</label>
    <input id="inline-email" type="email" placeholder="Email">

    <label for="inline-password">Password</label>
    <input id="inline-password" type="password" placeholder="Password">

    <label for="inline-remember">

    </label>

</form>

Ambient is a lightweight and modular HTML5 UI development methodology.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS).

Ambient makes use of HTML5 elements & CSS3 properties to layout the document outline.
