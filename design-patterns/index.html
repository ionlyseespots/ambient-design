Overview

Admin template | UI elements | UX shells | Navs | Form states

There are 4 distinct design patterns for layout when using Ambient.

Design patterns are built on Ambient's core fluid design system for HTML sections as a presentation format without the need for structure based classes.

CSS attribute selectors do not care if it is a predefined or a custom namespace. And it's the key to unlocking this new approach.

We define custom namespaces for design patterns. To avoid collisions with other namespaces, we are prefixing default design patterns with "ambient".

Introduction

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. There are 4 distinct design patterns for layout.

  1. UI Elements
  2. UX Shells (first priority)
  3. Navs
  4. Form States (second priority)
