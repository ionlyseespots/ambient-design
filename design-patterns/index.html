Layout attributes

Introduction

Ambient is layout

Ambient makes use of CSS3 attributes on elements.

We define our own namespaces

Because we define our own custom attributes, we can define extendable properties common to all HTML elements as well as semantic elements.

Ambient does not style components by default. View all components

Do use CSS3 attributes to layout defined content.
Do NOT use CSS3 attributes as a replacement for styling content better suited to a class name.

Namespaces

Global

Global attributes are layout properties common to all HTML elements. These attributes can be used on any element.

am-margin
am-padding
am-layer

Define your layout

The am-layout attribute must be used on a root semantic element, though the attribute may have no effect on some elements.

Global

Whitespace

Margin

To add space outside an element, add the attribute am-margin to any element.

If needed, additional size values large & small are available.

am-margin

If needed, specific direction is available as:

  • am-margin="top"
  • am-margin="right"
  • am-margin="bottom"
  • am-margin="left"

Padding

To add space inside an element, add the attribute am-padding to any element.

If needed, additional size values large & small are available.

am-padding

If needed, specific direction is available as:

  • am-padding="top"
  • am-padding="right"
  • am-padding="bottom"
  • am-padding="left"

Layers

To add a layer depth value (z-index), add the attribute am-layer="VALUE" to any element.

If needed, the additional skin classes for shadows .am-shadow-VALUE are available. *Shadows can be used on any element.

Layer depth values are available as

  • am-layer="z1"
  • am-layer="z2"
  • am-layer="z3"
  • am-layer="z4"
  • am-layer="z5"

Grid layout

Please note:

Grid layout is going away in 1.4.0 and is is being replaced with am-block View CodePen example

Grid layout is used to position content within your scaffolding, and thus relies on the root semantic element for intent.

The default orientation on portrait for grid layout does not automatically stack. The value portrait-100 can be added for defining a linear vertical layout for grid columns on portrait.

Grid layout with equal widths

To create a grid layout for rows and columns with equal widths, add the layout attributes am-block & am-block to unsemantic div elements.

Alternatively, if you do not need rows and simply want columns add the layout attribute am-block to the root semantic element. i.e., <section am-block>

Grid layout is used to position content within your scaffolding, and thus relies on the root semantic element for intent. The default orientation on portrait for grid layout does not automatically stack.

To add block, add the child attribute am-block to a div.
To add a column, add the child attribute am-block and define the offset width as am-block="VALUE" to a div. The default value (customizable as a less variable) for columns 12.

Grid layout with gutter

To create a grid layout with gutter, define the gutter & type by adding the values gutter and gutter TYPE as am-block="gutter TYPE".

Grid layout is used to structure content, and thus relies on the root semantic element for intent. The default orientation on portrait for grid layout does not automatically stack. The value portrait-100 can be added for defining a linear vertical layout for grid columns on portrait.

Mix & match column gutters

You can define any combination of layouts which enable you to have multiple types of grids

All columns in unsemantic div rows

  • am-block="gutter margin"
  • am-block="gutter padding"

All columns in semantic element rows:

  • am-layout="gutter margin"
  • am-layout="gutter padding"
