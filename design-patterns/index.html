Layout attributes

Introduction

Ambient is layout

Ambient makes use of CSS3 attributes on elements for page layout.

We define our own namespaces

Because we define our own custom attributes, we can define extendable properties common to all HTML elements as well as semantic elements.

Ambient does not style components by default. View all components

Do use CSS3 attributes to layout defined content.
Do NOT use CSS3 attributes as a replacement for styling content better suited to a class name.

Global attributes are layout properties common to all HTML elements. These attributes can be used on any element.

Global

Global layout attributes are layout properties common to all HTML elements. These attributes can be used on any element.

Margin

Default margin is 0
To add space outside an element, add the attribute am-margin to any element.

If needed, additional size values large & small are available.

am-margin

If needed, specific direction is available as:

  • am-margin="top"
  • am-margin="right"
  • am-margin="bottom"
  • am-margin="left"

Padding

Default padding is 0
To add space inside an element, add the attribute am-padding to any element.

If needed, additional size values large & small are available.

am-padding

If needed, specific direction is available as:

  • am-padding="top"
  • am-padding="right"
  • am-padding="bottom"
  • am-padding="left"

Layers

To add a layer and/or depth to an element, add the attribute am-layer="VALUE" to any element.

If needed, the additional value hvr-elevate and shadow is available.

Layer depth values are available as

  • am-layer="z1"
  • am-layer="z2"
  • am-layer="z3"
  • am-layer="z4"

Linear layout

Default

The HTML5 article element aligns the explicit sections section and aside in a single direction, vertically or horizontally. You can specify a vertical layout direction by adding the attribute and value am-layout="vertical" to the HTML5 article element. The default orientation is horizontal.

Horizontal

The HTML5 article element groups explicit sections (scales up to 12 equal sections) on one horizontal line. You're good to go.

Grid layout

Grid layout is used to structure content, and thus relies on the root semantic element for intent.

The default orientation on portrait for grid layout does not automatically stack. The value portrait-100 can be added for defining a linear vertical layout for grid columns on portrait.

Grid layout with equal widths

To create a grid layout with equal widths, add the layout attributes am-row & am-col to unsemantic div elements.

Alternatively, if you do not need rows and simply want columns add the layout attribute am-row to the root semantic element. i.e., <section am-row>

Grid layout is used to structure content, and thus relies on the root semantic element for intent. The default orientation on portrait for grid layout does not automatically stack. The value portrait-100 can be added for defining a linear vertical layout for grid columns on portrait.

To add a row, add the child attribute am-row to a div.
To add a column, add the child attribute am-col to a child div. The default for each column is equal and scales up to 12.

To add a row, add the child attribute am-row to a child div.
To add an offset width column, add the child attribute am-col and define the offset width as am-col="VALUE" to a child div.

Grid layout with gutter

To create a grid layout with gutter, define the gutter & type for the individual unsemantic row by adding the values gutter and gutter TYPE as am-row="gutter TYPE".

Alternatively, create a gutter for all content in an explicit section making use of the grid by adding am-layout="gutter TYPE" to the root semantic element.

Grid layout is used to structure content, and thus relies on the root semantic element for intent. The default orientation on portrait for grid layout does not automatically stack. The value portrait-100 can be added for defining a linear vertical layout for grid columns on portrait.

Mix & match gutters

Ambient allows your grid to have infinite combinations: a row of content can have no gutter, a gutter with margin or a gutter with padding.

Gutter type values are available as:

  • am-row="gutter margin"
  • am-row="gutter padding"

Alternatively, all columns in an explicit section can have no gutter, a gutter with margin or a gutter with padding.

  • am-layout="gutter margin"
  • am-layout="gutter padding"

Ambient is a lightweight and modular HTML5 UI development methodology.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS).

Ambient makes use of HTML5 elements & CSS3 attributes to layout the document outline.
