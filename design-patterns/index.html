Design patterns

Overview

Design patterns are built on Ambient's semantic grid.

While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic grid.

Introduction

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values.

Attribute selectors

It is possible to style HTML elements that have specific attributes and values, not just class and id.

We define our own namespaces.

am-component

am-layout

am-nav

am-form

Because we define our own custom attribute, we can prescribe base design patterns without needing to use or duplicate class names for layout.

Built with LESS to write well-structured, extendable code which is easy to maintain.

Infinitely nestable gutterless mixed grid layouts

This means you can consistently build UI into simple or complex layouts.

Summary

Attribute selectors select an element using the presence of a given attribute or attribute value.

[attr]
Represents an element with an attribute name of attr.
[attr=value]
Represents an element with an attribute name of attr and whose value is exactly "value".
[attr~=value]
Represents an element with an attribute name of attr whose value is a whitespace-separated list of words, one of which is exactly "value".
[attr|=value]
Represents an element with an attribute name of attr. Its value can be exactly “value” or can begin with “value” immediately followed by “-” (U+002D). It can be used for language subcode matches.
[attr^=value]
Represents an element with an attribute name of attr and whose value is prefixed by "value".
[attr$=value]
Represents an element with an attribute name of attr and whose value is suffixed by "value".
[attr*=value]
Represents an element with an attribute name of attr and whose value contains at least one occurrence of string "value" as substring.

Initial properties

Initial properties are margin, padding, and alignment which can be added to base design patterns. Base design patterns are: am-component, am-layout, am-nav, am-form.

Pattern matching is something I am yet working on for initial properties.

Margin

Grid margin is 0.

To add space outside, add the attribute am-margin to any base design pattern.

If needed, set a specific direction or size with the attribute and value as

Angular

By defining our own namespace for a design pattern, we are able to combine custom attributes + custom directive names.

<article foobar> </article >

[foobar] { //css }

We can use the directive name as the attribute selector for the custom design pattern. Now, when we use the angularJS directive "foobar" it will:

  1. inherit the Ambient semantic grid for <article>
  2. apply its own design pattern [foobar] { //css }
  3. apply any contextual overrides <article foobar="value"> </article > as individual design components (values) [foobar~="value"] { //css }

All without needing to use or duplicate class names for layout.

A Basic Directive

.directive('foobar', function() {
var obj = {
restrict: 'AC',
templateUrl: 'partials/foobar.html'
}
return obj;
});

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). Ambient avoids grid CSS classes in the HTML markup itself.

While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic grid.

Our goal is to deliver a lean, lightweight semantic grid for HTML elements, attributes and values, while also maintaining a separation from content that adapts across different devices.

Ambient makes use of HTML5 elements & CSS3 properties to convey a consistent visual structure within a semantic grid.

Design patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. And it's the key to unlocking this new approach.
