Getting started

Introduction

Ambient is semantic layout

HTML5 sectioning elements within Ambient define themselves, sectioning content in the document outline. This means you can build content into simple or complex layouts.

Bower

Bower: v1.2.2 


$ bower install ambient

This is the basic form of Ambient. 


ambient/
└── css/
    ├── ambient.css
    └── ambient.min.css

The css/ folder includes the semantic layout and UI patterns. The file comes as a single CSS file as well as a minified version.

Download

Bower: v1.2.3

Alternatively, get the ambient.min.css file from the repo and save the file in your directory.

Source

Bower: v1.2.3

Clone, fork or download the repo for complete source code, docs and this website. Ambient source files are in the less/ folder. Source code is the most current development (unstable) version of Ambient.

Back to basics

Ambient is ready to adopt whatever your design or style is. Ambient makes use of HTML5 elements & CSS3 properties to layout semantic markup.
Feel free to use semantic HTML5 tags, whenever and wherever you like.

Code 


<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<title>Ambient starting HTML5 example</title>
<!-- Ambient -->
<link href="ambient/css/ambient.min.css" rel="stylesheet">
</head>
<body>
<header role="banner">
   <h1>
        header
   </h1>
</header>
<hr />

<nav role="navigation">
    <a href="#">nav</a>
    <a href="#">nav</a>
    <a href="#">nav</a>
</nav>
<hr />

<main role="main">
    <article>
        <section>
            <h1>
                Hello, world!
            </h1>
        </section>
        <aside>
            <h1>
                aside
            </h1>
        </aside>
    </article>
</main>
<footer role="contentinfo">
    <p>
        sticky footer
    </p>
</footer>
</body>
</html>

Ambient is a lightweight and modular HTML5 UI development methodology.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). Ambient makes use of HTML5 elements & CSS3 properties to layout semantic markup.

Ambient makes use of HTML5 elements & CSS3 properties to layout the document outline. Ambient UI patterns are layout-related attributes for content.
