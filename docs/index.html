Download

Install with Bower 

$ bower install ambient

This is the basic form of Ambient. 

ambient/
└── css/
    ├── ambient.css
    └── ambient.min.css

The css/ folder includes the semantic layout and UI patterns. The file comes as a single CSS file as well as a minified version.

Source code

Clone or Fork the repo for complete source code, docs and this website.

Ambient source files are in the less/ folder.

Source code is the most current development (unstable) version of Ambient.

Ambient

am·bi·ent /'ambēənt/ adjective

  1. of the surrounding area or environment
  2. completely surrounding; encompassing
  3. creating a certain reaction or mood, often a subconscious one, by being wherever people tend to be
  4. pertaining to or noting sounds that create a peaceful and relaxed atmosphere
  5. pertaining to or noting close and constant social contact and communication fostered by the Internet or the use of digital devices

Starter template

Start with this starter HTML5 template, or modify our examples.

Copy the starter HTML5 template to begin a web project with Ambient, or modify our examples below. 

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<title>Starter HTML template</title>
<!-- Ambient -->
<link href="ambient/css/ambient.min.css" rel="stylesheet">
</head>
<body>
<header role="banner">
   <h1>Hello, header!</h1>
</header>
<hr />

<nav role="navigation">
    <a href="#">nav</a>
    <a href="#">nav</a>
    <a href="#">nav</a>
</nav>
<hr />

<main role="main">
    <article>
            <h1>Hello, world!</h1>
    </article>
</main>
<hr />

<footer role="contentinfo">
       <h1>Hello, sticky footer!</h1>
</footer>
</body>
</html>

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). Ambient avoids CSS classes in the HTML markup itself.

While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout

Our goal is to deliver a lean, lightweight semantic layout for HTML elements, attributes and values, while also maintaining a separation from content that adapts across different devices.

Ambient makes use of HTML5 elements & CSS3 properties to convey a consistent layout.

UI patterns are about defining namespaces for the HTML to live in. It works by generating styles for elements, attributes & values. And it's the key to unlocking this new approach.

This site is built with Ambient v1.1.3
