DO use HTML5 elements to layout semantic markup.

Do NOT use HTML5 elements as a replacement for divs.

Ambient is semantic layout.

Developers can focus on HTML5 elements to create layout and designers can focus on extending UI patterns for content.

Introduction

Ambient makes use of HTML5 elements & CSS3 properties to layout semantic markup. The CSS3 flexbox layout is defined by HTML5 elements. Here's how Ambient works:

Layout examples

Ambient is a method for creating semantic layout not constrained by the browser. HTML5 sectioning elements within Ambient define themselves, creating a visual structure for the document outline. This means you can build content into simple or complex layouts.

Ambient

Semantic Layout Description
body Column Represents the content.
header Row Base element which defines the header for a page or section.
nav Row Base element which groups navigation links on one horizontal line.
main 100% Defines the main or important content.
h1,h2,h3,h4,h5,h6 100% Briefly describes the topic of the section it introduces.
P 100% Represents a paragraph of text.
article Row Base element which groups content on one horizontal line. Note: on orientation portrait, the content will automatically stack.
section Column Base element which divides an article content into equal sections (columns) that scales up to 12 columns.
aside Column Base element which extends content into sections (columns) loosely related to the page.
footer Row Defines the footer for a page or section.
hr 100% Represents a thematic break and optionally defines framing.

HTML5

Sections

Element Description
<body>
Represents the content of an HTML document. There is only one <body> element in a document.
<section> Defines a section in a document.
<nav> Defines a section that contains only navigation links.
<article> Defines self-contained content that could exist independently of the rest of the content.
<aside> Defines some content loosely related to the page content. If it is removed, the remaining content still makes sense.
<h1>,<h2>,<h3>,<h4>,<h5>,<h6> Heading elements implement six levels of document headings; <h1> is the most important and <h6> is the least. A heading element briefly describes the topic of the section it introduces.
<header> Defines the header of a page or section. It often contains a logo, the title of the Web site, and a navigational table of content.
<footer> Defines the footer for a page or section. It often contains a copyright notice, some links to legal information, or addresses to give feedback.
<address> Defines a section containing contact information.
<main> Defines the main or important content in the document. There is only one <main> element in the document.

The HTML <hr> element represents a thematic break between paragraph-level elements (for example, a change of scene in a story, or a shift of topic with a section). In previous versions of HTML, it represented a horizontal rule. It may still be displayed as a horizontal rule in visual browsers, but is now defined in semantic terms, rather than presentational terms.

CSS3

Ambient is a CSS3 flexbox layout.

Normalize.css

For improved cross-browser rendering, we use Normalize.css, a project by Nicolas Gallagher and Jonathan Neal.

Flexbox LESS mixin

For improved flexbox rendering and maintainability, we use flexbox.less, a project by Jesper Johansen.

Cursor Reset

For improved cursor rendering, we use cursor-reset.css, a project by Tim Murtaugh.

Ambient is a methodology for creating layout.

This system evolved out of the need to fully decouple markup (HTML) from presentation (CSS). While presentation changes can require changes in structure, content can be changed without any need for structural change beyond the semantic layout.

Our goal is to deliver semantic layout for HTML elements, attributes and values, while also maintaining a separation from content from content that scales up from Portrait to Landscape.

Ambient layout makes use of HTML5 elements & CSS3 properties to layout semantic markup.

Ambient UI patterns are layout-related attributes for content.
