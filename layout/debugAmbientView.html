Do not think of heading tags as simple titles. They define a new, implicit, section when they aren't the first heading of their parent, explicit, sections.

Think of h1-h6 as dividers, or labels for the content below it. The content is defined by the heading, the heading does not end at the / it ends when it defines a new implicit section. I have found this tool very helpful and building a CSS version for am-layout Document Outliner
